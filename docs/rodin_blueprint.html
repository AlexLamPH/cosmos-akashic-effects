§2

Current Executive Summary

Hyper3D Rodin Gen-2.5 là một paid SaaS biến MỘT ảnh (hoặc tối đa 5 ảnh multi-view, hoặc một text prompt) thành một 3D model production-ready ở định dạng glb / usdz / fbx / obj / stl — chạy SERVER-SIDE, KHÔNG cần GPU phía Ra.

rodin3d-skills là Claude Code skill chính chủ của DeemosTech — một wrapper Python mỏng (~500 LOC: api_client.py / generate_3d_model.py / image_utils.py) bọc 3 endpoint tại api.hyper3d.com: /v2/rodin (submit) · /v2/status (poll) · /v2/download (fetch). Auth qua Bearer token, multipart upload, async task polling — không backend, không daemon, không telemetry.

Đây mở khóa đúng năng lực Ra HIỆN KHÔNG CÓ: 3D model GENERATION. Ra bây giờ chỉ RENDER mesh có sẵn qua Three.js — không tạo được mesh mới. Fit hoàn hảo với 3D leg Ra đã có: Rodin GENERATES file .glb → Three.js của Ra (GLTFLoader) + Playwright recordVideo RENDERS turntable video / marketing still / motion shot. Generation và render khớp thành một pipeline liền mạch.

Phase 1 & 2 (stack $0: GSAP + Remotion + design-system reference + watermark) đã ship + Polaris-APPROVED. Đây là MỘT món duy nhất cần paid key trong toàn bộ gear — đòn bẩy 3D lớn nhất, rẻ nhất, no-GPU.

Khuyến nghị thẳng: prototype TRƯỚC bằng test key miễn phí vibecoding built-in của Rodin ($0) để validate trọn pipeline image→glb→turntable, RỒI Alex provision một HYPER3D_API_KEY trả phí + một budget cap ($20–50/tháng, alert ở 80%). Không chi tiền trước khi pipeline đã chứng minh chạy.

Problem / Context

Ra là AI cloud / headless — không màn hình, mọi thứ drive bằng CLI / API / script. Ra render được nhưng không generate được 3D: Three.js (GLTFLoader) hiển thị mesh có sẵn, nhưng Ra không có cách tạo mesh mới từ ảnh hay text.

Blueprint gear-radar trước đó (Ra Phase-3 gear assessment) đã flag Rodin là món unlock paid có đòn bẩy cao nhất — vì nó cho 3D generation mà không cần GPU (chạy server-side của Hyper3D). Mọi tool GPU khác (WorldGen, VOID, WorldMirror) đều nặng và chưa có nhu cầu production cụ thể; Rodin một mình cover nhu cầu 3D-object thực tế.

Affected (đối tượng ảnh hưởng): năng lực sản xuất 3D của Ra. Đây là bước nâng cấp từ "render-only" lên "generate + render", không phải thử nghiệm rời rạc.

Current Proposal

Đưa Rodin vào làm GENERATION tier của 3D leg dưới hub AK-WFL-CAL0001, ghép thẳng với render tier (Three.js) đã có.

(a) Pipeline — generate → render liền mạch

  • INPUT — image / text / ≤5 multi-view image.
  • SUBMIT — rodin3d-skill multipart POST /v2/rodin (images + tier + format + Bearer auth) → server trả subscription_key + task_uuid.
  • POLL — client poll /v2/status tới khi task done.
  • DOWNLOAD — fetch result qua /v2/download → lưu .glb vào output dir.
  • RENDER (Ra) — Three.js GLTFLoader load .glb → turntable qua Playwright recordVideo.
  • OUTPUT — on-brand turntable video / marketing still / motion shot (webm/MP4).
